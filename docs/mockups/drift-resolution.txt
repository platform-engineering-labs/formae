║                  RECONCILE REJECTED — TUI MOCKUPS                           ║
║                                                                              ║
║  Shown when formae apply is rejected because stacks have been modified       ║
║  since the last reconcile (FormaReconcileRejectedError).                     ║
║                                                                              ║
║  Current behavior: prints extract commands + manual delete instructions.     ║
║  New behavior: interactive view showing out-of-band changes with deltas,     ║
║  multi-select for extraction, or revert all with --force.                    ║
╚══════════════════════════════════════════════════════════════════════════════╝

Data source: FormaReconcileRejectedError
  - ModifiedStacks: map[stackName] → ModifiedStack
    - ModifiedResources: []ResourceModification
      - Stack, Type, Label, Operation
      - PatchDocument (JSONPatch of the drift)                   [NEW - API addition]

Operations in ResourceModification:
  - "update" — resource properties changed out-of-band
  - "create" — resource appeared out-of-band (not in forma code)
  - "delete" — resource disappeared out-of-band (still in forma code)

User actions (at the bottom, not per-resource):
  - Extract selected  — extract the selected resources as code into a file
  - Revert all        — re-apply with --force (reverts ALL out-of-band changes)
  - Abort             — exit, user handles it manually

================================================================================
API CHANGES NEEDED
================================================================================

ResourceModification needs one new field:

  ResourceModification struct {
    Stack         string          `json:"Stack"`
    Type          string          `json:"Type"`
    Label         string          `json:"Label"`
    Operation     string          `json:"Operation"`
    PatchDocument json.RawMessage `json:"PatchDocument,omitempty"` // NEW
  }

  - PatchDocument: JSONPatch document describing the out-of-band change
    (same format as ResourceUpdate.PatchDocument). Reuses the existing
    FormatPatchDocument rendering pipeline.
  - For "update": PatchDocument populated with the property diff
  - For "create": no PatchDocument (just label/type/stack shown)
  - For "delete": no PatchDocument (resource is gone from reality,
    still in forma code — user must manually remove to accept)
