================================================================================
DETECTION LOGIC
================================================================================

At startup:
  1. Check Sixel support (query terminal with escape sequence, short timeout)
  2. Check Kitty graphics protocol support
  3. If neither: check if terminal supports braille (Unicode block 0x2800-0x28FF)
  4. Fallback: plain text

The logo rendering is encapsulated in a function that returns a string,
so components just call RenderLogo(size) and get the right version.

Size variants:
  - Full: used by non-TUI commands (banner before output)
  - Compact: used in TUI panel headers
  - None: when terminal is too narrow or graphics not supported
